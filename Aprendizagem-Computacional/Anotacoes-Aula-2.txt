Apresentação dos Pilares da Aprendizagem Computacional

1) Decomposição 
Um problema muito difícil significa que existe um problema menor a ser solucionado.
A primeira fase dessa estratégia é a análise que consiste estudar e explorar o problema a fim de que você consiga decompor os elementos e realizar as etapas de maneira mais detalhada.
A etapa de síntese consiste em combinar os elementos a fim de remontar o problema original para que haja sentido na solução proposta para trazer coerência.

Ordem de execução das tarefas menores:

a) Sequencial: dependência ou ordem entre os problemas menores fazendo com que seja necessário seguir um passo a passo em fila;
b) Paralelo: realizar as tarefas ao mesmo tempo paralelisando o problema. É importante no pensamento computacional pois diminui o tempo e aumenta a eficiência;

A decomposição tras as ferramentas de pequenos problemas, identificação de variáveis e segmentação. 
Não basta aplicar a decomposição  mas sim é necessário realiza-la sozinho sem utilizar modelos propostos.
	-> entender o problema
	-> identificar componentes
	-> identificar características e dependências

2) Reconhecimento de Padrões
Padrão: modelo de referência que permite construir e reconstruir um objeto a partir de uma referência de algo estruturado. 
Possui uma estrutura invariante com características bem definidas. 

O reconhecimento de padrões busca identificar a estrutura invariante a partir da similiraridade e indiferenças do modelo em questão
	ex: armazenamento de fotos de redes sociais possui o padrão de compressão para realizar a operação nos servidores

O reconhecimento de padrões é trivial para o ser humano, ou seja, simplesmente fazemos isso.

É importante determinar padrões pois é por meio dessa técnica há a generalização e estrapolamento. 
Os objetos são classificados em classes e categorias variando do domínio e do tipo de mídia a ser tratada.

Como o computador faz? 
O computador reconhece padrões por meio de:
	a) Dadis que representem atributos;
	b) Aprendizado - entendendo os conceitos associados aos objetos;
	c) Armazenar os dados para que o computador os acesse e tome a decisão de classificação;
	d) Regras de decisão;
Conceito muito utilizado em redes neurais

Resumindo, o reconhecimento de padrões parte de uma arbodagem onde as características do problema são extraidas, classifica-se os dados obtidos utilizando metodos distintos e, com as análise, esses dados são aplicados em diversos contextos que se assemelhem

Utilizado em:
	-> Classificação de dados
	-> Reconhecimento de Imagem
	-> Análise de cenas
	-> Classificação de documentos
	-> Reconhecimento de fala
As principais áreas que se beneficiam disso são as áreas de machine learning, redes neurais, inteligência artificial e ciência de dados.

3) Abstração
Abstrair: ato de abstrair que consiste em observar diferentes elementos avaliando suas características individuais.

Abstração: processo intelectual de isolamento de um objeto da realidade, transformando-o em algo generalista tirando sua dependência do concreto e possibilitando utiliza-lo em diversos contextos.

Generalizar: tornar algo geral, mais amplo, extendê-lo. Possibilitando a utilização em diversas situações. Pela lógica é uma operação que consiste reunir uma classe geral, um conjunto de seres ou fenômenos similares.

	!lembrar de variaveis como X

Para classificar os dados é necessário identificar seus pontos essenciais para generalizar os objetos a fim de classifica-los para conseguir uma representação e constextualizar.

	!Merge sort: classificação

Esse conceito é utilizado nas áreas de arquitetura de redes, computadores, comunicação, algoritmos entre outros diversos da área da computação.

ex: por onde começar a limpar o terreno? 
	identificar as características para encontrar a primeira etapa para a solução.
	 

4) Algoritmos
Um dos pilares mais importantes pois é nele que é possível definir o passo a passo da solução.
Determinante para executar as intruções. 
As instruções são passadas para os computadorores por meio dos usuários. 
O principal objetivo de um computador é receber dados e instruções, trata-los e armazena-los.

Um algoritmo é um processo de resolução de problemas passo a passo utilizando as intruções passadas pelo programa/usuario. Elas trazem as ordens de execução das instruções e o que elas vão fazer

No desenvolvimento de um software existem três etapas sendo elas: 
	a) Análise: estudo e definição dos dados de entrada e saída
	b) Algoritmo: descreve o problema por meio de ferramentas narrativas, fluxogramas ou pseudocodigos para estruturar o algoitmo e determinar as instruções detalhadas;
	c) Codificação: o algoritmo é trazido para uma linguagem de programação;

Um algoritmo é uma sequência de passos bem definidos com um objetivo, execução de tarefas específicas e um conjunto de operações que resultam em uma sucessão finita de ações.

	ex: cozinha uma lasanha.

Para construir um algoritmo primeiro é necessário a compreensão do problema, depois definir os dados de entrada. Em sequência, é necessário definir qual o processamento, quais os dados de saída, utilizar métodos de construção e por fim realizar testes e diagnósticos. 

Os principais métodos para essa construção são: 
	a) Narrativa: mais simples e intuitivo pois usa a linguagem natural do ser humano para definir as instruções passo a passo sem a utilização de conceitos novos. Pode apresentar diversas insterpretações diferentes;
	b) Fluxograma: utilização de simbolos pré definidos para exemplificar a linguagem do computador;
	c) Pseudocódigo: utilização de uma linguagem que lembra uma linguagem de programação porem numa escrita mais simples porém com regras bem definidas. É mais parecido com a codificação em si;
